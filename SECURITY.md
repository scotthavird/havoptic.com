# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Havoptic, please report it responsibly:

1. **Do not** open a public GitHub issue
2. Email the maintainer directly or use GitHub's private vulnerability reporting feature
3. Include details about the vulnerability and steps to reproduce

## Response Timeline

- We will acknowledge receipt within 48 hours
- We will provide an initial assessment within 7 days
- We will work with you to understand and resolve the issue

## Scope

This security policy applies to:
- The Havoptic website (havoptic.com)
- This GitHub repository

## Out of Scope

- Third-party services and APIs we integrate with
- Issues in dependencies (please report those to the respective projects)

## Recognition

We appreciate security researchers who help keep Havoptic safe. With your permission, we will acknowledge your contribution in our release notes.
